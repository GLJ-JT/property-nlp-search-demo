05 Narrative

The full case in five short sections.

1. Problem

Property seekers rarely think in neat filters. A real query sounds more like: studio near UCL under 2500, bills included, lift, quiet, not basement. Standard portals make users translate that sentence into separate controls, then remember what the system may have missed. I treated the problem as confidence leakage: if users cannot see which constraints were understood, they create their own workaround through screenshots, notes, agent messages, saved links, and repeated searches.

2. Diagnosis

The core question was not whether natural-language search could parse words. It was whether the interface could prove its interpretation. Users needed to know which requirements were hard filters, which were soft preferences, and which ones needed correction. That changed the direction from an AI search box to an explainable intent layer: keep the original query visible, turn parsed meaning into chips, explain ranking, and offer recovery when the request is too strict or ambiguous.

3. Build

The frontend handles query state, chips, filters, onboarding, result cards, and saved searches. The Python backend parses intent, separates hard filters from soft signals, ranks listings, and returns plain-language reasons. The JSON dataset stores rent, stations, landmarks, amenities, accessibility, and derived tags. I kept the stack deliberately readable so the product logic can be inspected, challenged, and improved.

4. Validation

The wireframe was tested with 10 users on realistic renter tasks: find a studio near a landmark, stay under budget, exclude unwanted listing types, edit one parsed filter, and explain why the top result matched. Average task completion moved from 6m20s to 3m45s, query reformulations dropped from 3.1 to 1.4 per task, shortlist success rose from 60% to 85%, and confidence rose from 5.8 to 8.1 out of 10. These are prototype results, not production analytics, but the behavioural pattern was clear.

5. Next

The project shows a working product system: interaction model, parser, ranking, data, demo, and commercial sizing. The next step would be a controlled pilot on real renter traffic. I would measure saved properties, contact clicks, reformulation rate, return sessions, and lead quality. That would show whether explainable NLP belongs in search, saved alerts, onboarding, or premium agent tools.
